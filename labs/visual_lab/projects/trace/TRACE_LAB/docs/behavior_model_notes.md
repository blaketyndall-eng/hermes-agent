# TRACE Behavior Model Notes

TRACE should model behaviors, not aesthetics.

The distinction is important.

Aesthetics are surface-level styles.
Behaviors are recurring visual tendencies.

---

# Example

Weak direction:

```text
"Make it look vintage."
```

Stronger direction:

```text
"Preserve dimensional shadows while allowing slight environmental instability."
```

The second instruction is:

- observable
- measurable
- reproducible
- reviewable

---

# TRACE Core Idea

TRACE should eventually become:

```text
behavior layering
```

Instead of:

```text
single preset transforms
```

---

# Proposed Layers

## Scene behavior

What the environment is doing.

Examples:

- sodium vapor spill
- fluorescent overheads
- dark apartment corners
- flash collapse
- wet pavement reflections

---

## Camera behavior

How the camera interprets the scene.

Examples:

- shadow retention
- falloff
- slight softness
- restrained texture
- instability

---

## Circulation behavior

How the image changes over time digitally.

Examples:

- screenshot softness
- repost degradation
- resized uploads
- compression residue

---

## Roll behavior

How a sequence of images feels coherent.

Examples:

- stable tonal tendencies
- repeated darkness behavior
- consistent environmental feel
- subtle variation without preset repetition

---

# Most Important Principle

TRACE should avoid:

- loud signatures
- obvious stylistic identity
- dramatic transformations

The strongest TRACE behavior may be:

emotionally recognizable
without being technically recognizable.
