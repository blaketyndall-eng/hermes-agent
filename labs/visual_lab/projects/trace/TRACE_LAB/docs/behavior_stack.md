# TRACE Behavior Stack

TRACE should process images through layered behavioral systems.

Avoid:

- giant monolithic filters
- aesthetic presets
- all-at-once transformations

Prefer:

small layered behavior systems.

---

# Proposed Stack

```text
scene
→ camera
→ circulation
→ roll
→ evaluation
```

Each layer contributes small believable tendencies.

---

# 1. Scene Layer

The environment being captured.

Examples:

- fluorescent convenience store
- sodium vapor parking lot
- apartment kitchen
- dark dive bar
- rainy sidewalk

Possible behaviors:

- mixed_light_ambiguity
- environmental_darkness
- reflective_highlights
- foreground_pressure
- physical_falloff

---

# 2. Camera Layer

How TRACE interprets the environment.

Possible behaviors:

- darkness_confidence
- shadow_texture
- motion_softness
- restrained_instability
- dimensional_black

---

# 3. Circulation Layer

How digital movement affects the image.

Possible behaviors:

- screenshot_softness
- repost_damage
- compression_residue
- archive_feeling

---

# 4. Roll Layer

How multiple images behave together.

Possible behaviors:

- roll_drift
- tonal_consistency
- environmental_continuity
- restrained_variation

---

# 5. Evaluation Layer

Human review and comparison.

Possible systems:

- paired comparison
- rubric scoring
- behavior tagging
- memory realism review
- failure logging

---

# Important Principle

TRACE should not become:

"a recognizable look"

The strongest behavior stack may feel:

emotionally coherent
without being technically obvious.
