# TRACE Drift Execution Notes

Controlled drift should feel psychologically invisible.

The goal is not:

- visible variation
- random instability
- dramatic sequence movement

The goal is:

```text
subtle lived inconsistency
```

---

# Key Principle

Drift should slightly influence behavior.

It should not override:

- scene reality
- environmental continuity
- emotional coherence

---

# Good Drift

Examples:

- slightly deeper darkness late in a sequence
- subtle texture fluctuation
- small motion softness increase
- restrained environmental adaptation

Good drift feels:

- human
- physical
- emotionally invisible

---

# Bad Drift

Examples:

- random tonal swings
- obvious contrast shifts
- visible texture jumps
- inconsistent atmosphere
- preset-like alternation

Bad drift feels:

- artificial
- mechanical
- algorithmic

---

# Important Observation

The viewer should rarely notice drift directly.

Instead, drift should contribute to:

- sequence realism
- anti-phone behavior
- emotional continuity
- memory realism

---

# Current Implementation Strategy

Current system:

```text
base profile
→ drift offsets
→ drifted profile
→ parameter mapping
→ image execution
```

This keeps:

- behavior inspectable
- execution modular
- sequence logic understandable

---

# Long-Term Direction

Eventually drift may also respond to:

- time of night
- environmental transitions
- motion intensity
- flash usage
- sequence pacing
- emotional density

But TRACE should remain restrained.
