# TRACE Environment Signals

TRACE should eventually respond to environmental conditions.

But environment understanding should remain:

- inspectable
- behavior-oriented
- lightweight
- grounded in observable image characteristics

Avoid:

- black-box scene classifiers
- vague aesthetic tagging
- generic AI labels

---

# Current Signal System

The current system extracts:

```text
darkness_level
shadow_coverage
highlight_concentration
contrast_level
color_ambiguity
```

These are not scene labels.

They are:

behavioral clues.

---

# 1. darkness_level

Measures overall image darkness.

Potential influence:

- darkness confidence
- shadow preservation
- memory pressure

---

# 2. shadow_coverage

Measures how much of the image is under shadow.

Potential influence:

- dimensional black handling
- foreground pressure
- anti-phone behavior

---

# 3. highlight_concentration

Measures isolated bright regions.

Potential influence:

- memory anchors
- environmental contrast
- highlight weighting

---

# 4. contrast_level

Measures luminance spread.

Potential influence:

- shadow crush restraint
- falloff
- environmental continuity

---

# 5. color_ambiguity

Measures disagreement between color channels.

Potential influence:

- mixed-light preservation
- environmental ambiguity
- anti-normalization behavior

---

# Important Principle

Signals should:

- gently influence behavior
- never fully determine behavior

TRACE should remain:

scene-responsive
without becoming deterministic.

---

# Long-Term Direction

Eventually TRACE may build:

```text
environment signals
→ behavioral tendencies
→ sequence adaptation
→ memory realism optimization
```

But the system should remain subtle and interpretable.
