# TRACE Profile Execution Notes

Profiles should influence behavior.

They should not behave like presets.

---

# Key Principle

A profile is:

- a weighted behavioral environment
- a collection of tendencies
- a guide for experimentation

A profile is NOT:

- a fixed visual look
- a static LUT
- a deterministic style transform

---

# Example

```text
night_walk_profile
```

does not mean:

"make every image look identical"

It means:

- preserve darkness
- allow restrained instability
- favor dimensional shadows
- preserve environmental ambiguity

while still adapting to the actual scene.

---

# Profiles Should Influence

Examples:

- shadow depth
- texture restraint
- motion softness
- foreground pressure
- falloff
- circulation residue
- roll drift

---

# Profiles Should Not Override Reality

Important:

The captured scene should remain primary.

TRACE should avoid:

- forcing visual identity
- destroying environmental truth
- applying obvious signatures

The strongest TRACE behavior may:

subtly guide the image
without visibly taking control of it.

---

# Long-Term Goal

Eventually:

```text
profile
→ influences experiment execution
→ influences roll behavior
→ influences evaluation expectations
```

This creates:

coherent emotional tendencies
without preset repetition.
